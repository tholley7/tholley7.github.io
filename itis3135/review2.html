Peer Review 2 - Noah Parkes's Project

File Management/Naming

The files have capital letters that would be more appropriate in Java code. They are snake case (e.g. "donationPage.html"); instead, they should be named something like "donations.html" or "donations_page.html" to conform with this class's naming conventions. The fonts used in the site are files rather than css imports and are named "bodyFont.woff2" and "headingFont.woff2". These should be properly named according to the actual fonts' names.

There are no folders in the project directory. All files (html, css, and woff2) are in the same top-level directory. There are also no images in the project directory, leading me to believe that they are located in the class site's images directory. Each file, other than the HTML files, should be in corresponding folders according to document type. Additionally, the project's images should be in a folder within the project directory.

Styles

The site is made up of default styles. There CSS document (communityStyleSheet.css) in the project directory is not properly implemented in the head of each page's HTML. This should be an easy fix.

Page Design

The page lacks any custom design due to the failed css file import.

The footer does have copyright that attempts to link to the design firm page but the link is broken. This is probably due to a misformatted file path.

One good thing is that each page has the same organizational structure. The brand title is at the top in large bold text, followed by the nav bar, a smaller heading with the page title, and finally the body text/image content.

Validation

There are no validation buttons or links.

Page Code

The page code is properly indented and all ids and classes appear intentional. Each page's content is contained within a div with the same class ("content") which is the only thing inside the "main" tag. This "content" div is unnecessary as all of it's content would fit inside "main" with no issue.
